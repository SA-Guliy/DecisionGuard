## Fail-Safe Publish Rule
- Final sign-off uses the physical publish root only: `github_publish/`.
- Blocking audit validates that on-disk publish files exactly match `PUBLISH_EXPORT_MANIFEST.json` (path+sha256).
- Any unlisted file, missing file, or hash mismatch is `FAIL` with non-zero exit.
- Blocking pre-push control: `git add demo_sources/investor_demo/*` must be executed before final push/commit.
